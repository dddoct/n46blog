# 紫团博客图片爬虫 - 功能说明

## 基于 https://www.nogizaka46.com/s/n46/diary/MEMBER

## 📋 核心命令功能

| 命令 | 功能说明 | 示例 |
|------|---------|------|
| `crawl` | 模式1：爬取博客列表和图片信息 | `python main.py crawl --pages 5` |
| `download` | 模式1：下载所有图片 | `python main.py download` |
| `full` | 模式1：完整流程（爬取+下载） | `python main.py full --pages 10` |
| `member` | 模式2：指定成员检索 | `python main.py member --name "池田 瑛紗"` |
| `stats` | 显示统计信息 | `python main.py stats` |
| `export` | 导出博客数据为JSON | `python main.py export` |

---

## 🔧 命令选项

| 选项 | 说明 | 默认值 | 适用命令 |
|------|------|--------|---------|
| `--pages N` | 限制爬取页数 | 无限制(crawl)/3(member) | `crawl`, `full`, `member` |
| `--name "成员名"` | 指定成员日文名 | 必填 | `member` |

---

## 🎯 两种检索模式详解

### 模式1：总检索

**用途**：全量备份所有成员的博客和图片

**特点**：
- 爬取所有成员的博客
- 三种分类存储方式
- 输出到 `output1/` 目录
- 适合首次全量爬取

**命令**：
```bash
# 爬取前10页博客
python main.py crawl --pages 10

# 下载图片
python main.py download

# 或一键完成
python main.py full --pages 10
```

**输出结构**：
```
output1/
├── data/
│   └── n46_blog.db          # SQLite数据库
└── images/
    ├── by_member/           # 按成员分类
    ├── by_date/             # 按日期分类
    └── by_blog/             # 按博客分类
```

### 模式2：指定成员检索

**用途**：针对特定成员的个人博客页面进行定向爬取

**特点**：
- 只爬取指定成员的博客
- 使用成员个人页面API
- 输出到 `output2/` 目录
- 按日期细分子目录
- 适合追踪特定成员

**命令**：
```bash
# 默认3页
python main.py member --name "池田 瑛紗"

# 指定5页
python main.py member --name "小川 彩" --pages 5
```

**输出结构**：
```
output2/images/
└── 池田 瑛紗/
    ├── 20241231/
    │   ├── IkedaTeresa_1.png
    │   └── IkedaTeresa_2.jpg
    └── 20250101/
        ├── IkedaTeresa_1.jpg
        └── IkedaTeresa_2.png
```

## 📊 数据管理功能

### 1. 博客数据爬取
- 从API获取博客列表
- 解析博客内容提取图片URL
- 支持增量爬取（不会重复记录图片）

### 2. 图片下载
- 异步并发下载（默认5个并发）
- 自动重试机制（失败时重试3次）
- 智能分类存储：

#### 模式1 - 三种分类方式

**by_member（按成员分类）**
- 目录结构：`output1/images/by_member/成员日文名/`
- 命名格式：`英文_日期_序号.jpg`
- 示例：`by_member/小川 彩/OgawaAya_20260301_1.jpg`

**by_date（按日期分类）**
- 目录结构：`output1/images/by_date/年月/日期/`
- 命名格式：`英文_序号.jpg`
- 示例：`by_date/202601/20260101/OgawaAya_1.jpg`

**by_blog（按博客分类）**
- 目录结构：`output1/images/by_blog/博客ID/`
- 命名格式：`序号_原文件名`
- 示例：`by_blog/12345/1_mobq2EkxC.jpg`

#### 模式2 - 成员专属分类

**成员日期分类**
- 目录结构：`output2/images/成员名/日期/`
- 命名格式：`英文_序号.jpg`
- 示例：`output2/images/池田 瑛紗/20241231/IkedaTeresa_1.jpg`

### 3. 数据持久化
- SQLite数据库存储博客和图片元数据
- 记录下载状态（已下载/待下载）
- 防止重复插入同一图片

### 4. 成员名字映射
- 日文名转英文名
- 支持成员名字变体（带空格、特殊字符等）
- 可手动扩展映射表（members.py）

## 🚀 使用流程

### 首次使用（推荐）

1. **安装依赖**
   ```bash
   pip install -r requirements.txt
   ```

2. **模式1：全量爬取**
   ```bash
   python main.py full --pages 20
   ```

3. **查看统计**
   ```bash
   python main.py stats
   ```

### 日常更新

- **增量更新**：运行 `python main.py crawl --pages 5` 获取最新博客
- **下载新图片**：运行 `python main.py download`

### 追踪特定成员

```bash
# 获取池田瑛紗最近3页的博客
python main.py member --name "池田 瑛紗"

# 获取小川彩最近5页的博客
python main.py member --name "小川 彩" --pages 5
```

## ⚙️ 配置参数

在 `config.py` 中可以修改：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `DOWNLOAD_DELAY` | 请求间隔(秒) | 1 |
| `MAX_RETRIES` | 下载失败重试次数 | 3 |
| `TIMEOUT` | 请求超时时间(秒) | 30 |
| `CONCURRENT_DOWNLOADS` | 并发下载数 | 5 |

## 🔍 常见问题

**Q: 模式2找不到成员怎么办？**
A: 先运行一次模式1爬取成员信息，或检查成员名字是否正确。

**Q: 如何只下载特定成员的图片？**
A: 使用模式2 `member` 命令，会自动下载该成员的所有图片。

**Q: 两种模式的图片会重复吗？**
A: 不会，因为存储在不同的目录（output1/ 和 output2/）。

**Q: 可以修改默认页数吗？**
A: 可以，模式2默认3页，可以通过 `--pages` 参数修改。
