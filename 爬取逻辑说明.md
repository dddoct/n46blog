## 数据流说明

### 阶段1：爬取博客信息

```
API请求 (list/blog)
    │
    ▼
解析JSONP响应
    │
    ▼
提取博客数据 ──► 保存到数据库 (blogs表)
    │                ├─► 新博客：INSERT
    │                └─► 已存在：UPDATE (INSERT OR REPLACE)
    ▼
提取图片URL ────► 保存到数据库 (images表)
                     ├─► 新图片：INSERT (download_status = 0)
                     └─► 已存在：跳过 (UNIQUE约束)
```

### 阶段2：下载图片

```
查询待下载图片 (download_status = 0)
    │
    ▼
按博客ID分组，计算图片序号
    │
    ▼
异步下载图片 ──► 保存到三个目录
    │              ├─► by_member/成员名/英文_日期_序号.jpg
    │              ├─► by_date/年月/英文_序号.jpg
    │              └─► by_blog/博客ID/原文件名
    ▼
更新数据库状态 (download_status = 1, local_path = 路径)
```

## 数据库设计

### blogs 表（博客信息）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT PRIMARY KEY | 博客唯一ID |
| title | TEXT | 博客标题 |
| author | TEXT | 作者名（日文） |
| author_id | TEXT | 作者ID |
| publish_date | TEXT | 发布日期 |
| url | TEXT | 博客URL |
| content | TEXT | 博客内容（HTML） |
| created_at | TIMESTAMP | 记录创建时间 |

**关键特性**：使用 `INSERT OR REPLACE`，相同ID会更新整条记录

### images 表（图片信息）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PRIMARY KEY AUTOINCREMENT | 自增ID |
| blog_id | TEXT | 所属博客ID |
| original_url | TEXT | 原始图片URL |
| local_path | TEXT | 本地保存路径 |
| filename | TEXT | 文件名 |
| file_size | INTEGER | 文件大小（字节） |
| width | INTEGER | 图片宽度 |
| height | INTEGER | 图片高度 |
| download_status | INTEGER DEFAULT 0 | 下载状态（0=未下载，1=已下载） |
| created_at | TIMESTAMP | 记录创建时间 |

**约束**：`UNIQUE(blog_id, original_url)` - 防止同一图片重复记录

## 覆盖逻辑详解

### 1. 博客数据的覆盖逻辑

```python
# database.py
self.cursor.execute('''
    INSERT OR REPLACE INTO blogs (id, title, author, ...)
    VALUES (?, ?, ?, ...)
''', (...))
```

| 场景 | 行为 | 结果 |
|------|------|------|
| 博客ID不存在 | INSERT | 插入新记录 |
| 博客ID已存在 | REPLACE | 更新所有字段 |

**实际效果**：
- 重复爬取同一篇博客会更新其内容
- 适合追踪博客的编辑历史

### 2. 图片数据的覆盖逻辑

```python
# database.py
try:
    self.cursor.execute('INSERT INTO images ...')
    self.conn.commit()
except sqlite3.IntegrityError:
    # (blog_id, original_url) 组合已存在
    # 返回已存在的记录ID，不插入新记录
    return result[0]
```

| 场景 | 行为 | 结果 |
|------|------|------|
| (blog_id, url) 不存在 | INSERT | 插入新记录，status=0 |
| (blog_id, url) 已存在 | 跳过 | 不插入，返回已有ID |

**实际效果**：
- 同一图片不会被重复记录到数据库
- 即使博客内容被更新，已记录的图片不会重复

### 3. 文件下载的覆盖逻辑

```python
# scraper.py
with open(filepath, 'wb') as f:
    f.write(content)
```

| 场景 | 行为 | 结果 |
|------|------|------|
| 文件不存在 | 创建新文件 | 写入内容 |
| 文件已存在 | 覆盖写入 | 替换原有内容 |

**实际效果**：
- 每次下载都会覆盖同名文件
- 修改命名规则后，需要重置下载状态才能重新生成

## 使用场景示例

### 场景1：首次完整爬取

```bash
# 爬取前5页博客信息
python main.py crawl --pages 5

# 下载所有图片
python main.py download
```

**结果**：
- 数据库：新建 blogs 和 images 记录
- 文件：生成 by_member、by_date、by_blog 三个目录

### 场景2：增量更新（新增博客）

```bash
# 再次执行爬取
python main.py crawl --pages 5

# 下载新增图片
python main.py download
```

**结果**：
- 数据库：
  - 已有博客：更新内容（REPLACE）
  - 新增博客：插入记录
  - 已有图片：跳过（不重复插入）
  - 新增图片：插入记录（status=0）
- 文件：只下载新增图片

### 场景3：修改命名规则后重新生成

```bash
# 1. 重置下载状态
python reset_download.py

# 2. 删除旧文件（可选）
Remove-Item -Recurse output/images/by_member/*
Remove-Item -Recurse output/images/by_date/*

# 3. 重新下载（使用新命名规则）
python main.py download
```

**结果**：
- 数据库：所有图片 status 重置为 0
- 文件：按新规则重新生成，覆盖旧文件

### 场景4：完全重新开始

```bash
# 1. 删除数据库
Remove-Item output/data/n46_blog.db

# 2. 删除所有图片
Remove-Item -Recurse output/images/*

# 3. 重新爬取和下载
python main.py full --pages 5
```

**结果**：
- 完全清空所有数据，重新开始

## 关键代码片段

### 数据库保存博客

```python
def save_blog(self, blog_data):
    """保存博客信息 - 已存在的会更新"""
    self.cursor.execute('''
        INSERT OR REPLACE INTO blogs (id, title, author, author_id, 
                                      publish_date, url, content)
        VALUES (?, ?, ?, ?, ?, ?, ?)
    ''', (...))
    self.conn.commit()
```

### 数据库保存图片

```python
def save_image(self, image_data):
    """保存图片信息 - 如果已存在则跳过"""
    try:
        self.cursor.execute('''
            INSERT INTO images 
            (blog_id, original_url, local_path, filename, file_size, width, height)
            VALUES (?, ?, ?, ?, ?, ?, ?)
        ''', (...))
        self.conn.commit()
        return self.cursor.lastrowid
    except sqlite3.IntegrityError:
        # 图片已存在，返回已存在的记录ID
        self.cursor.execute('''
            SELECT id FROM images WHERE blog_id = ? AND original_url = ?
        ''', (blog_data['blog_id'], blog_data['original_url']))
        result = self.cursor.fetchone()
        return result[0] if result else None
```

## 注意事项

1. **数据库文件位置**：`output/data/n46_blog.db`

## 总结

| 操作 | 数据库行为 | 文件行为 |
|------|-----------|----------|
| 爬取博客 | INSERT OR REPLACE | 无 |
| 爬取图片 | INSERT（跳过重复） | 无 |
| 下载图片 | UPDATE status | 覆盖写入 |
| 重置状态 | UPDATE status=0 | 无 |

这种设计确保了：
- ✅ 不会重复记录同一图片
- ✅ 可以更新博客内容
- ✅ 支持增量爬取
- ✅ 可以强制重新生成文件
