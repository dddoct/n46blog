# 紫团博客图片提取 - 功能说明

## 核心命令功能

| 命令 | 功能说明 | 示例 |
|------|---------|------|
| `full` | 模式1：完整流程（爬取+下载） | `python main.py full --pages 2` |
| `member` | 模式2：指定成员检索 | `python main.py member --name "池田 瑛紗"` |
| `crawl` | 仅爬取博客列表 | `python main.py crawl --pages 3` |
| `download` | 仅下载图片 | `python main.py download` |
| `stats` | 显示统计信息 | `python main.py stats` |
| `export` | 导出博客数据为JSON | `python main.py export` |

---

## 命令选项

| 选项 | 说明 | 默认值 | 适用命令 |
|------|------|--------|---------|
| `--pages N` | 限制爬取页数 | 1 (full) / 3 (member) | `full`, `member` |
| `--name "成员名"` | 指定成员日文名 | 必填 | `member` |

---

## 两种检索模式详解

### 模式1：总检索

**用途**：使用 Selenium 遍历总博客页面，爬取所有成员的最新博客

**特点**：
- 使用 Chrome 浏览器访问 `https://www.nogizaka46.com/s/n46/diary/MEMBER?ima=5900&page=N`
- 支持真正的分页（每页约 30-40 篇博客）
- 三种分类存储方式（by_member、by_date、by_blog）
- 输出到 `output1/` 目录
- 从博客详情页获取准确日期

**命令**：
```bash
# 爬取1页（约30-40篇博客）
python main.py full

# 爬取2页
python main.py full --pages 2

# 爬取5页
python main.py full --pages 5
```

**输出结构**：
```
output1/
├── data/
│   └── n46_blog.db          # SQLite数据库
└── images/
    ├── by_member/           # 按成员分类
    ├── by_date/             # 按日期分类
    └── by_blog/             # 按博客分类
```

### 模式2：指定成员检索

**用途**：针对特定成员的个人博客页面进行定向爬取

**特点**：
- 使用 requests 访问成员个人页面
- 每页约 10 篇博客
- 按日期细分子目录
- 输出到 `output2/` 目录
- 从博客详情页获取准确日期

**命令**：
```bash
# 默认3页
python main.py member --name "池田 瑛紗"

# 指定5页
python main.py member --name "小川 彩" --pages 5
```

**输出结构**：
```
output2/images/
└── 池田 瑛紗/
    ├── 20241231/
    │   ├── IkedaTeresa_1.png
    │   └── IkedaTeresa_2.jpg
    └── 20250101/
        ├── IkedaTeresa_1.jpg
        └── IkedaTeresa_2.png
```

## 数据管理功能

### 1. 博客数据爬取
- 从网页获取博客列表
- 访问每篇博客详情页提取准确的发布日期
- 解析博客内容提取图片URL
- 支持增量爬取（不会重复记录图片）

### 2. 图片下载
- 异步并发下载（默认5个并发）
- 自动重试机制（失败时重试3次）
- 智能分类存储

### 3. 数据存储
- SQLite 数据库记录博客和图片元数据
- 支持断点续传
- 下载状态跟踪

## 分类存储说明

### 模式1 - 三种分类方式

#### by_member（按成员分类）
- 目录结构：`output1/images/by_member/成员日文名/`
- 命名格式：`英文成员名_日期_序号.jpg`
- 示例：`by_member/小川 彩/OgawaAya_20260301_1.jpg`

#### by_date（按日期分类）
- 目录结构：`output1/images/by_date/年月/日期/`
- 命名格式：`英文成员名_序号.jpg`
- 示例：`by_date/202601/20260101/OgawaAya_1.jpg`

#### by_blog（按博客分类）
- 目录结构：`output1/images/by_blog/博客ID/`
- 命名格式：`序号_原始文件名.jpg`
- 示例：`by_blog/104302/1_mobq2EkxC.jpg`

### 模式2 - 按成员和日期分类

- 目录结构：`output2/images/成员名/日期/`
- 命名格式：`英文成员名_序号.jpg`
- 示例：`output2/images/池田 瑛紗/20241231/IkedaTeresa_1.png`

## 日期获取说明

由于紫团官网的博客列表页面不显示发布日期，日期只在博客详情页才有，因此爬虫采用以下策略：

1. **列表页**：获取博客ID、标题、作者、URL
2. **详情页**：访问每篇博客详情页，从 `.bd--hd__date` 等元素中提取准确日期
3. **数据更新**：使用详情页获取的日期更新数据库记录

这样可以确保：
- `by_member` 中的图片文件名包含正确日期（如 `OgawaAya_20260301_1.jpg`）
- `by_date` 按正确的年月日分类（如 `by_date/202603/20260301/`）

## 使用建议

### 首次全量爬取
```bash
# 爬取多页（如10页，约300-400篇博客）
python main.py full --pages 10
```

### 定期增量更新
```bash
# 每天爬取1-2页最新博客
python main.py full --pages 1
```

### 追踪特定成员
```bash
# 爬取指定成员的最新博客
python main.py member --name "池田 瑛紗" --pages 3
```
