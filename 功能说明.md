# 乃木坂46博客图片爬虫 - 功能说明

## 📋 核心命令功能

| 命令 | 功能说明 | 示例 |
|------|---------|------|
| `crawl` | 爬取博客列表和图片信息 | `python main.py crawl --pages 5` |
| `download` | 下载所有待下载的图片 | `python main.py download` |
| `full` | 完整流程（爬取+下载） | `python main.py full --pages 10` |
| `stats` | 显示统计信息 | `python main.py stats` |
| `export` | 导出博客数据为JSON | `python main.py export` |

---

## 🔧 命令选项

| 选项 | 说明 | 适用命令 |
|------|------|---------|
| `--pages N` | 限制爬取页数 | `crawl`, `full` |

## 📊 数据管理功能

### 1. 博客数据爬取
- 从API获取博客列表
- 解析博客内容提取图片URL
- 支持增量爬取（不会重复记录图片）

### 2. 图片下载
- 异步并发下载（默认5个并发）
- 自动重试机制（失败时重试3次）
- 三种分类存储：

#### by_member（按成员分类）
- 目录结构：`by_member/成员日文名/`
- 命名格式：`英文_日期_序号.jpg`
- 示例：`by_member/小川 彩/OgawaAya_20260301_1.jpg`

#### by_date（按日期分类）
- 目录结构：`by_date/年月/`
- 命名格式：`英文_序号.jpg`
- 示例：`by_date/202603/OgawaAya_1.jpg`

#### by_blog（按博客分类）
- 目录结构：`by_blog/博客ID/`
- 命名格式：`ID_原文件名`
- 示例：`by_blog/12345/1_original.jpg`

### 3. 数据持久化
- SQLite数据库存储博客和图片元数据
- 记录下载状态（已下载/待下载）
- 防止重复插入同一图片

### 4. 成员名字映射
- 日文名转英文名
- 支持成员名字变体（带空格、特殊字符等）
- 可手动扩展映射表

## ⚙️ 技术特性

- **API-based爬取**：直接调用官方API，非HTML解析
- **异步下载**：使用aiohttp实现高效并发下载
- **数据去重**：数据库唯一约束防止重复记录
- **错误处理**：网络错误自动重试，超时保护
- **灵活配置**：支持修改并发数、超时时间等参数

## 📝 命令详细说明

### crawl 命令
爬取博客列表和图片信息，保存到数据库。

```bash
# 爬取所有页面
python main.py crawl

# 只爬取前5页
python main.py crawl --pages 5
```

**特点**：
- 已存在的博客会更新内容
- 已存在的图片不会重复插入
- 可以随时中断，下次继续

### download 命令
下载所有待下载的图片（download_status = 0）。

```bash
python main.py download
```

**特点**：
- 只下载未下载过的图片
- 异步并发下载，速度快
- 自动保存到三个分类目录

### full 命令
完整流程：先爬取博客，然后下载图片。

```bash
# 爬取所有页面并下载
python main.py full

# 爬取前10页并下载
python main.py full --pages 10
```

**特点**：
- 一站式完成爬取和下载
- 适合首次使用或完整更新

### stats 命令
显示当前数据统计信息。

```bash
python main.py stats
```

**输出示例**：
```
==================================================
📊 数据统计
==================================================
  博客总数:     100
  成员数:       32
  图片总数:     309
  已下载图片:   304
  待下载图片:   5
==================================================
```

### export 命令
将博客数据导出为JSON文件。

```bash
python main.py export
```

**输出**：`output/data/blogs_export.json`

## 🔍 数据库说明

### blogs 表（博客信息）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT PRIMARY KEY | 博客唯一ID |
| title | TEXT | 博客标题 |
| author | TEXT | 作者名（日文） |
| author_id | TEXT | 作者ID |
| publish_date | TEXT | 发布日期 |
| url | TEXT | 博客URL |
| content | TEXT | 博客内容（HTML） |
| created_at | TIMESTAMP | 记录创建时间 |

### images 表（图片信息）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PRIMARY KEY AUTOINCREMENT | 自增ID |
| blog_id | TEXT | 所属博客ID |
| original_url | TEXT | 原始图片URL |
| local_path | TEXT | 本地保存路径 |
| filename | TEXT | 文件名 |
| file_size | INTEGER | 文件大小（字节） |
| width | INTEGER | 图片宽度 |
| height | INTEGER | 图片高度 |
| download_status | INTEGER DEFAULT 0 | 下载状态（0=未下载，1=已下载） |
| created_at | TIMESTAMP | 记录创建时间 |

**约束**：`UNIQUE(blog_id, original_url)` - 防止同一图片重复记录

## ⚠️ 注意事项

1. **数据库文件位置**：`output/data/n46_blog.db`
2. **图片存储位置**：`output/images/`
3. **成员映射表**：`members.py`（可手动添加新成员）
4. **重置下载状态**：运行 `python reset_download.py`
5. **并发下载数**：默认5个并发，可在 `config.py` 中修改

## 🆘 常见问题

### Q: 如何重新开始爬取？
A: 删除数据库文件和所有图片：
```bash
Remove-Item output/data/n46_blog.db
Remove-Item -Recurse output/images/*
python main.py full --pages 5
```

### Q: 如何只下载特定成员的图片？
A: 目前不支持按成员筛选下载，但可以通过文件系统筛选：
```bash
# 查看特定成员的图片
ls "output/images/by_member/小川 彩/"
```

### Q: 下载失败了怎么办？
A: 重新运行下载命令即可，会自动重试失败的图片：
```bash
python main.py download
```

### Q: 如何添加新成员的英文名映射？
A: 编辑 `members.py` 文件，在 `MEMBER_NAME_MAP` 字典中添加：
```python
"新成员日文名": "NewMemberEnglishName",
```
