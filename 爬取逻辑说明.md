## 双模式设计

### 模式1：总检索（使用 Selenium）

**目标页面**：`https://www.nogizaka46.com/s/n46/diary/MEMBER?ima=5900&page=N`

**为什么使用 Selenium？**

1. **页面动态加载**：
   - 直接请求 HTML 返回空页面（没有博客数据）
   - 数据通过 JavaScript 调用 API 动态渲染
   - API 本身不支持分页参数（始终返回最新100篇）

2. **解决方案**：
   - 使用 Selenium 启动 Chrome 浏览器
   - 访问页面并等待 JavaScript 执行完成
   - 解析渲染后的 HTML 提取博客数据
   - 实现真正的分页爬取

**技术实现**：

## 日期获取逻辑

### 问题背景

紫团官网的博客列表页面**不显示发布日期**，日期信息只在博客详情页才有。

### 解决方案

采用"列表页+详情页"的两阶段爬取策略：

```
列表页获取基本信息
    │
    ├─ 博客ID
    ├─ 标题
    ├─ 作者
    └─ URL
    │
    ▼
访问详情页
    │
    ├─ 提取发布日期（.bd--hd__date）
    ├─ 提取博客内容
    └─ 提取图片URL
    │
    ▼
更新数据库
```

### 日期提取代码

```python
def _fetch_blog_detail(self, url):
    # ... 获取页面 ...
    soup = BeautifulSoup(html, 'html.parser')
    
    # 提取发布日期
    publish_date = None
    date_selectors = [
        '.bd--hd__date',  # 主要选择器
        '.bd--ne__date',
        'time',
        '.date',
    ]
    for selector in date_selectors:
        date_elem = soup.select_one(selector)
        if date_elem:
            date_text = date_elem.get_text().strip()
            # 解析日期格式 2026.03.02 19:54
            date_match = re.search(r'(\d{4}[./-]\d{2}[./-]\d{2})', date_text)
            if date_match:
                date_str = date_match.group(1)
                for fmt in ['%Y.%m.%d', '%Y/%m/%d', '%Y-%m-%d']:
                    try:
                        dt = datetime.strptime(date_str, fmt)
                        publish_date = dt.strftime('%Y/%m/%d %H:%M:%S')
                        break
                    except:
                        continue
    
    return {
        'content': content,
        'images': images,
        'publish_date': publish_date  # 从详情页获取的日期
    }
```

### 日期更新


## 数据库设计

### blogs 表（博客信息）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT PRIMARY KEY | 博客唯一ID |
| title | TEXT | 博客标题 |
| author | TEXT | 作者名（日文） |
| author_id | TEXT | 作者ID |
| publish_date | TEXT | 发布日期（从详情页获取） |
| url | TEXT | 博客URL |
| content | TEXT | 博客内容（HTML） |
| crawled_at | TIMESTAMP | 爬取时间 |

### images 表（图片信息）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PRIMARY KEY AUTOINCREMENT | 自增ID |
| blog_id | TEXT | 所属博客ID |
| original_url | TEXT | 原始URL |
| local_path | TEXT | 本地存储路径 |
| download_status | INTEGER | 0=未下载, 1=已下载 |
| created_at | TIMESTAMP | 创建时间 |

**约束**：
- `UNIQUE(blog_id, original_url)`：防止重复记录同一图片

## 关键技术点

### 1. Selenium 分页实现

**问题**：总博客页面使用 JS 动态加载，直接请求 HTML 拿不到数据

**解决**：
```python
def crawl_blog_list(self, start_page=1, max_pages=None):
    driver = self._create_selenium_driver()
    for page in range(start_page, max_pages + 1):
        blogs = self._parse_total_blog_page(driver, page)
        # 处理博客数据...
    driver.quit()
```

### 2. 日期获取策略

**问题**：列表页没有日期，详情页才有

**解决**：
- 两阶段爬取：列表页获取基本信息 → 详情页获取日期
- 使用 CSS 选择器 `.bd--hd__date` 提取日期
- 支持多种日期格式解析

### 3. 图片去重机制

**数据库层面**：
```sql
UNIQUE(blog_id, original_url)
```

**代码层面**：
```python
processed_blogs = set()
for blog in blogs:

    processed_blogs.add(blog['id'])
```

### 5. 成员ID获取

**方式1**：从数据库查找（如果模式1已爬取过）
**方式2**：从 API 实时查找

```python
def _get_member_id(self, member_name):
    # 先尝试从数据库获取
    self.db.cursor.execute(
        'SELECT DISTINCT author_id FROM blogs WHERE author LIKE ? LIMIT 1',
        (f'%{member_name}%',)
    )
    result = self.db.cursor.fetchone()
    if result and result[0]:
        return result[0]
    
    # 从 API 获取成员列表
    data = self.fetch_api("list/member")
    for item in data['data']:
        if member_name in item.get('name', ''):
            return item.get('arti_code')
```
