## 系统概述

本系统用于自动爬取紫团成员博客中的图片，支持两种操作模式，提供图形界面和命令行两种使用方式。

### 核心功能

- 自动爬取博客列表和图片URL
- 异步并发下载图片
- 智能分类存储（成员/日期/博客）
- 支持断点续传
- 实时日志显示

---

## 操作模式

### 模式1：总检索

**用途**：爬取紫团官网所有成员的最新博客

**特点**：
- 从官网博客列表页面爬取
- 每页约30-40篇博客
- 图片分类存储在 `output1/` 目录
- 支持三种分类方式：按成员、按日期、按博客ID

**适用场景**：
- 想获取所有成员的最新博客
- 需要按日期或成员分类整理

### 模式2：指定成员检索

**用途**：针对特定成员的个人博客页面进行深度爬取

**特点**：
- 从成员个人博客页面爬取
- 每页约10篇博客
- 图片分类存储在 `output2/` 目录
- 按成员名+日期两级分类

**适用场景**：
- 只关注特定成员
- 需要获取该成员的所有历史博客

## 使用方式

### 方式一：GUI图形界面（推荐）

双击运行 `n46blog_gui.exe`

**界面元素**：

| 元素 | 说明 |
|------|------|
| 模式选择 | 单选按钮切换模式1/模式2 |
| 爬取页数 | 设置要爬取的页面数量 |
| 成员名字 | 模式2时输入日文名 |
| 开始爬取 | 启动爬取流程 |
| 停止 | 中断当前爬取 |
| 查看统计 | 显示数据库统计信息 |
| 打开输出目录 | 根据当前模式打开对应目录 |
| 日志窗口 | 实时显示运行日志 |
| 进度条 | 显示爬取进度 |

**操作步骤**：

1. **模式1（总检索）**：
   - 选择"模式1：总检索"
   - 设置页数（建议1-5页）
   - 点击"开始爬取"
   - 等待完成

2. **模式2（成员检索）**：
   - 选择"模式2：指定成员检索"
   - 输入成员日文名（如："池田 瑛紗"）
   - 设置页数（建议3-5页）
   - 点击"开始爬取"
   - 等待完成

### 方式二：命令行

运行 `n46blog.exe`

```bash
# 模式1：总检索
n46blog.exe full --pages 2

# 模式2：指定成员检索  
n46blog.exe member --name "池田 瑛紗" --pages 3

# 查看统计
n46blog.exe stats

# 查看帮助
n46blog.exe --help
```

## 文件命名规则

### 模式1文件名格式

```
{英文姓}{英文名}_{日期}_{序号}.jpg

示例：
IkedaTeresa_20260301_1.jpg
OgawaAya_20260228_3.jpg
```

### 模式2文件名格式

```
{英文姓}{英文名}_{日期}_{序号}.jpg

示例：
IkedaTeresa_20241231_1.jpg
IkedaTeresa_20241231_2.jpg
```

### 命名规则说明

- **英文姓/名**：根据日文名自动转换
- **日期**：从博客详情页提取的发布日期（格式：YYYYMMDD）
- **序号**：同一博客中多张图片的序号

## 命令行使用

### 命令列表

```bash
# 完整流程（爬取+下载）
n46blog.exe full [--pages N]

# 仅爬取博客列表
n46blog.exe crawl [--pages N]

# 仅下载图片
n46blog.exe download

# 指定成员检索
n46blog.exe member --name "成员名" [--pages N]

# 查看统计
n46blog.exe stats

# 导出数据
n46blog.exe export [--format csv|json]
```

### 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--pages N` | 爬取页数 | 1 |
| `--name "xxx"` | 成员日文名 | 无 |
| `--format xxx` | 导出格式 | csv |

### 使用示例

```bash
# 模式1：爬取2页
n46blog.exe full --pages 2

# 模式2：爬取池田瑛纱3页
n46blog.exe member --name "池田 瑛紗" --pages 3

# 模式2：爬取小川彩
n46blog.exe member --name "小川 彩" --pages 5

# 查看数据库统计
n46blog.exe stats
```

## 常见问题

### Q1: 首次运行很慢？

**A**: 首次运行会自动下载 ChromeDriver，需要几秒钟时间，请耐心等待。

### Q2: 图片下载失败？

**A**: 可能原因：
- 网络连接问题
- 图片链接失效
- 磁盘空间不足

系统会自动重试3次，失败的图片会记录在数据库中。

### Q3: 如何重新下载失败的图片？

**A**: 运行下载命令即可：
```bash
n46blog.exe download
```

### Q4: 支持哪些成员名字格式？

**A**: 需要使用日文全名，如：
- "池田 瑛紗" ✓
- "小川 彩" ✓
- "池田瑛紗" ✗（缺少空格）
- "Ikeda Teresa" ✗（英文）

### Q5: 爬取会重复吗？

**A**: 不会。系统会检查数据库，已存在的博客和图片不会重复爬取。

### Q6: 可以中断后继续吗？

**A**: 可以。随时点击"停止"按钮中断，下次运行会自动跳过已下载的内容。

### Q7: 输出目录在哪里？

**A**: 
- 模式1：`程序所在目录/output1/`
- 模式2：`程序所在目录/output2/`

### Q8: 为什么有些日期显示"unknown"？

**A**: 正常情况下不应该出现。如果发生，可能是：
- 博客详情页解析失败
- 网页结构变化

建议更新到最新版本。
